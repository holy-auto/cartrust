## 1. セキュリティ — 重大欠陥 (5.5/10)

### CRITICAL: middleware.ts が存在しない

プロジェクト全体に**Next.js middleware が一切存在しない**。ルート保護はクライアントサイドのReactコンポーネント（`AdminRouteGuard`等）に完全依存しており、**サーバーサイドでのページ保護がゼロ**。

| ポータル | 保護方式 | リスク |
|---------|---------|-------|
| `/admin` | クライアント側 AdminRouteGuard | 認証チェック前にページが一瞬表示される |
| `/agent` | クライアント側 AgentRouteGuard | 同上 |
| `/insurer` | **ガードなし** | **保護機構が完全に欠落** |
| `/customer` | Cookie検証（遅延） | 認証前にUIが表示される |

### CRITICAL: Resend Webhook シークレット未検証

`/api/webhooks/resend/route.ts` — `RESEND_WEBHOOK_SECRET`が未設定の場合、**署名検証なしでWebhookを受け入れる**。本番環境で偽造イベントを受け付ける可能性。

### HIGH: Cron署名検証の実装バグ

`/lib/cronAuth.ts` — Vercel cron署名の検証で**空文字列をハッシュ**している。署名検証として機能していない。

### HIGH: テナントアクセス制御の不備

一部APIルートで`tenant_id`をパラメータとして受け取るが、呼び出し元がそのテナントの所有者であるか検証していない。LINEのWebhookルートが典型例。

### その他のセキュリティ懸念

- CSRF保護がNext.jsデフォルトに依存（明示的トークンなし）
- Admin APIルートにレートリミットが未適用
- 顧客ポータルの認証が「メール＋電話下4桁」のみ（脆弱）
- API鍵ローテーション戦略が未策定

### 評価できる点

- Zod によるバリデーション基盤は堅実
- Supabase RLS によるテナント分離は適切
- 公開エンドポイントにはレートリミットあり
- CSP/HSTS/X-Frame-Optionsなどのセキュリティヘッダーは適切
- Stripe Webhook署名検証は正しく実装
- ファイルアップロードのマジックバイト検証あり

## 2. DB/APIレイヤー — 不整合多数 (6.0/10)

### トランザクション非対応

Supabase PostgREST SDKは**トランザクションをサポートしない**。代理店承認（4〜5ステップ）やPOS決済など、複数テーブルを更新する処理が**非原子的**。中間ステップで失敗すると孤立データが発生する。

### APIレスポンス形式が不統一

247のAPIルートで**4種類以上のレスポンス形式**が混在：

```
Format A: { "agents": [...] }
Format B: { "ok": true, "item": {...} }
Format C: { "public_id": "...", ... }
Format D: { "error": "unauthorized" }
```

クライアント側の実装が脆弱になる原因。

### ページネーション未実装のエンドポイント

- `/api/admin/agents` — 全件返却（ページネーションなし）
- `/api/admin/invoices` — ページネーション不明
- `/api/agent/rankings` — ページネーションなし
- 20以上のGETエンドポイントでページネーション欠落

### バリデーション欠落

- 23以上のPOST/PUTルートでJSONパースエラーのtry-catchなし
- 注文作成、ドキュメント作成、請求書エンドポイントにZodスキーマ未適用
- 金額フィールドの型チェック不足

### その他のDB問題

- `job_orders.from_tenant_id = to_tenant_id`を防ぐ制約なし（自己参照注文が可能）
- 証明書テーブルの非正規化データ（車両情報）に同期トリガーなし
- ソフトデリート戦略なし（ほとんどのテーブルに`deleted_at`カラムがない）
- マイグレーションのロールバック手段なし（97マイグレーション全てが追記のみ）
- `pending_remote.sql`（6038バイト）が未コミット状態で残存

## 3. UI/フロントエンド — マーケティング未完成 (6.5/10)

### CRITICAL: マーケティングサイトが5ページ中5ページ「Coming Soon」

4/1ローンチに向けて**集客の入り口が存在しない**：

| ページ | 状態 |
|--------|------|
| `/pricing` (料金プラン) | **Coming Soon** |
| `/for-shops` (施工店向け) | **Coming Soon** |
| `/for-agents` (代理店向け) | **Coming Soon** |
| `/for-insurers` (保険会社向け) | **Coming Soon** |
| `/faq` (よくある質問) | **Coming Soon** |

トップページ (`/`) のみ実装済みだが、ポータルへの導線以外に**サービス説明・機能紹介・料金が一切ない**。

### SEOメタデータ欠落

管理画面含め100ページ以上でメタデータ未設定。マーケットプレイス商品ページ (`/market/[id]`) にも`generateMetadata()`なし。

### 型安全性の劣化

- `any`型が288箇所（129ファイル）で使用
- ESLintで`no-explicit-any`がwarn設定（エラーではない）
- 特にInsurer検索ページ（11箇所）、証明書新規作成（8箇所）が深刻

### UI一貫性の欠如

- ボタンスタイル：`btn-primary`、生のTailwind文字列、インラインgradientが混在
- スペーシング：`p-4`, `p-6`, `p-8`が規則なく使用
- カラー：CSS変数とハードコード(`bg-[#0f1117]`)が混在
- ステータスラベルの重複定義が複数ファイルに散在

### 未最適化の画像

- 2箇所で`<img>`タグを使用（Next.js `Image`コンポーネント未使用）
- サムネイル画像にalt属性が空

### エラーハンドリング

- `/app/market/error.tsx`のみSentry連携が欠落（他は全て連携済み）
- `.catch(() => {})`による無音エラー処理が複数箇所に存在
